最近加了个云南的鲜花群, 想着看看怎么回事儿, 养花到底是个什么感觉

整束花按自己的节奏展开, 见证了这个过程, 人也跟着静下来一些. 

这周没有什么大的事, 反而是这种小而确定的动作更让我有感觉.

按部就班地照料一点点小事, 再看结果按预期长出来, 这件事本身就很能治愈人. 花是这样, 做事也是这样. 

这周也在整理自己的 workflow. 与其继续往前堆更多零散脚本和规则, 不如把高频场景收成一个更稳, 更少淌坑, 也更容易复用的框架. 现在还是在打磨阶段, 但方向已经比之前清楚不少. 

比起马上追求做得更多, 现在更在意的是先把节奏感找回来. 什么该先做, 什么值得固化, 什么其实不用急着解决, 慢慢开始有了一点轮廓. 希望再过几周, 这套东西能整理到可以开源的程度. 

## 看到的东西

https://ngiokweng.github.io

![](/weekly/2616/Appdome.png)

写得很细, 尤其是 Appdome 分析那篇, 从里面学到了一些逆向时值得留意的处理手法. 比起具体写了什么, 我可能更喜欢他整个人传出来的感觉, 尤其是那句签名: 如果你对目前拥有的一切觉得不满, 等到你拥有更多时, 也不见得会快乐. 

这句话和我这周的状态挺搭. 很多时候问题不只是缺了什么, 而是节奏已经乱了.


https://github.com/DetachHead/rebased

![](/weekly/2616/rebased.png)

Jetbrains 的独立 git 管理工具, 之前 JetBrains 一直说要做, 后来又没有下文了, 结果有人把它开源复现出来了.

解压到 Applications 要 1.4GB 左右, 对一个 git 工具来说有点太大了, 但是管理那种 submodule 多的任务还是比纯命令行舒服很多.


https://github.com/tw93/kaku

![](/weekly/2616/kaku.png)

**Kaku is a deeply customized fork of WezTerm, designed for an out-of-the-box experience.**

在 X 上看到有人评价 tw93 写工具像真正的 craftsman. 确实, 平时看很多信息都会让人焦虑, 唯独看他做产品和聊产品, 会有一种日拱一卒, 乐在其中的感觉. 甚至睡前看一会儿还有点安神...

很打动我的点: **`vim` 快捷键配置**, **`sudo` 指纹验证**, **自定义配置模型调用 AI** (Warp 一直不开放这个事)

另一个印象很深的评价, 大意是说, 这是他十年来第一次用到一个几乎不用先折腾配置, 就已经觉得顺手的终端. 

我这倒是在我常用的命令上遇到了几个问题, 后面提一下


[android-cli](https://android-developers.googleblog.com/2026/04/build-android-apps-3x-faster-using-any-agent.html)

![](/weekly/2616/android-cli.png)

移动安全开发里, 很多固定检测场景以前都要手搓一个小应用. 现在 Google 推出 Android CLI 之后, 写好 plan 让他做编写测试闭环就够了, 感觉传统的 android 大前端开发也快到头了...


https://github.com/dugongzi/JsxposedX

![](/weekly/2616/Jsxposed.png)

把常见需求抽象成一个能被普通人直接调用的接口, 感觉学到了, 说一句话可以禁掉一个应用的弹窗

## 随便写写

最近看到张朝阳讲行为心理学, 里面有一段话让我很有感觉: 思考可以设计行为, 行为可以改变情绪. 

"**思考可以设计行为, 行为可以改变情绪, 盯着你自己, 然后设计, 去做你该做的事情的行为, 这个行为去实践之后, 就可以改变你的情绪**"

很多时候不是等我先想通了, 情绪先变好了, 我才开始做事. 更有效的方法可能是反过来, 先盯着自己, 先设计行为, 先去做那些该做的事情, 情绪再慢慢跟上. 

这和给花换水, 和整理 workflow, 其实是同一种方法. 先做那些小而确定的动作, 再等状态一点点回来.
